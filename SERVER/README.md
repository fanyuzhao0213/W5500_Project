# W5500 OTA Server

基于 Python + PyQt5 的 MQTT OTA 升级服务器，用于向 W5500 设备推送固件升级。

## 功能特性

### 核心功能
- ✅ **MQTT 连接管理**：支持配置 Broker 地址、端口、设备 ID
- ✅ **固件文件支持**：支持 HEX 和 BIN 两种格式的固件文件
- ✅ **自动分块传输**：支持 1KB/2KB/4KB/8KB 分块大小配置
- ✅ **Base64 编码**：数据传输前自动进行 Base64 编码
- ✅ **CRC32/MD5 校验**：固件完整性校验
- ✅ **实时进度显示**：可视化传输进度
- ✅ **日志记录**：完整的操作日志和状态记录

### 扩展功能
- 🎨 **深色主题**：现代化的深色 UI 界面
- 📱 **多设备管理**：支持多个设备的选择和切换
- ⚙️ **高级配置**：可配置传输延迟、分块大小等参数
- 💾 **配置保存**：自动保存上次配置，下次启动自动加载
- 📤 **日志导出**：支持将日志保存到本地文件
- 🔄 **线程安全**：MQTT 通信在独立线程中运行

## 使用方法

### 1. 连接 MQTT Broker
1. 在 **Connection** 标签页中配置：
   - **Broker Address**：MQTT 服务器地址（默认：47.74.187.120）
   - **Port**：MQTT 端口（默认：1883）
   - **Device ID**：目标设备 ID（如：w5500_001）
2. 点击 **Connect** 按钮连接

### 2. 选择固件文件
1. 切换到 **OTA Upgrade** 标签页
2. 点击 **Browse** 按钮选择固件文件（支持 .hex 或 .bin）
3. 输入固件版本号

### 3. 配置升级参数
- **Chunk Size**：分块大小（1KB/2KB/4KB/8KB）
- **Transfer Delay**：传输延迟（10-500ms）
- **Auto Retry**：失败自动重试

### 4. 开始升级
1. 点击 **Start Upgrade** 按钮
2. 观察进度条和日志信息
3. 等待升级完成提示

### 5. 查看日志
- 切换到 **Log** 标签页查看详细日志
- 支持自动滚动、显示时间戳
- 可保存日志到本地文件

## MQTT 主题

| 主题 | 方向 | 说明 |
|------|------|------|
| `device/{device_id}/ota/cmd` | 发布 | 升级命令（ota_start/ota_stop） |
| `device/{device_id}/ota/data` | 发布 | 固件数据块 |
| `device/{device_id}/ota/status` | 订阅 | 设备升级状态 |
| `device/{device_id}/ota/ack` | 订阅 | 数据块确认 |

## 协议格式

### 升级命令 (ota_start)
```json
{
    "cmd": "ota_start",
    "version": "1.0.0",
    "size": 49152,
    "crc32": 1234567890,
    "chunks": 12
}
```

### 数据块
```json
{
    "index": 0,
    "total": 12,
    "size": 4096,
    "data": "base64_encoded_data"
}
```

### 状态响应
```json
{
    "stage": 5,
    "progress": 100,
    "error": 0
}
```

**Stage 含义**：
- 0: IDLE（空闲）
- 1: CHECKING（检查中）
- 2: DOWNLOADING（下载中）
- 3: VERIFYING（校验中）
- 4: INSTALLING（安装中）
- 5: SUCCESS（成功）
- 6: FAILED（失败）

## 注意事项

1. 确保 MQTT Broker 可访问
2. 固件文件大小建议不超过 Flash 空间
3. 网络不稳定时建议增大传输延迟
4. 升级过程中请勿断开连接
5. 升级完成后设备会自动重启
